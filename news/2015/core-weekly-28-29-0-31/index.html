PrestaShop Core Weekly - Weeks 28-29-30-31 of 2015

An inside look at PrestaShop's codebase

Written by
Xavier Borderie
Open Source Advocate
Published

Aug 6, 2015

Share this

This edition of the Core Weekly report highlights changes in PrestaShop core codebase during the last four weeks, from Monday 6th of July to Sunday 2nd of August, 2015.

Wow, is this report late! We’re almost ready to enter the Core Monthly limit! So let’s dive right into it. Thank you for your patience!

Milestones

Important note: A security upgrade was released last week, for PrestaShop 1.4, 1.5 and 1.6! Version 1.6.1.0 is safe, but if you have any other version, please apply the patch! Read all about it in the blogpost, or find more technical details and how to apply the fix manually in this text file.

Now that PrestaShop 1.6.1.0 has been released, the team is working on the next version of your preferred e-commerce solution. In the meantime, we are planning on releasing version 1.6.1.1, a patch version that will fix some lingering issues from 1.6.1.0. It is slated for a late-August release, and you can test the first RC right now!

Want to contribute your code skills to the PrestaShop project? Make sure your Git setup is correctly configured!

We reached 1500 forks on GitHub! Don’t want to fork us but you really like the project? Star us on GitHub!

Of note among the merged PRs since the last Core Weekly Report:

  • In PR 3446, contributor kpodemski made it so that the “Transplant module” page shows the discription of the hook.
  • In PR 3511, contributor prestamodule changed the size of the modules’ icons so that it would be the same as on PrestaShop Addons (57*57).
  • In PR 3472, contributor tucoinfo added Indonesian states.
  • In PR 3503, contributor mcdado added zoom on Android devices.
  • In PR 3461, contributor PhpMadman added the possibility to display package items on order.

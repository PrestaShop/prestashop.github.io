PrestaShop Core Weekly - Week 8 of 2016

An inside look at PrestaShop's codebase

Written by
Xavier Borderie
Open Source Advocate
Published

Mar 1, 2016

Share this

This edition of the Core Weekly report highlights changes in PrestaShop’s core codebase during the last week, from Monday 22nd to Sunday 28th of February 2016.

Milestones

Two PRs to notice this week:

  • 5040: Introducing an OrderPresenter controller, by @mickaelandrieu.
  • 5069: introduced core js for themes, by @mickaelandrieu.

The team is hard at work on PrestaShop 1.7, the next major version of the most complete e-commerce software solution there is. You can find an up-to-date thread about it in the forum.

Please note that finishing the planned projects for 1.7 and delivering a quality beta version soon is the top priority for the PrestaShop Core developers at the moment. Merging community pull requests has therefore slowed down until then, but will be back with a vengeance once we have a first beta to release to you! So please keep testing, and keep sending patches our way: it’s the raison d’être of this community project!

Do you want to encourage us? Give us a star on GitHub! :)

Code changes in 1.7.0.0-develop

Let’s review the merged pull-requests on the ‘develop’ branch since the last Core Weekly Report!

Text changes

  • 5063: Update Italian installer and Hong Kong localization pack, by @maximebiloe. Thank you @mcdado for the original italian PR!

Product page

  • 5042: Add pack mention, by @maximebiloe.
  • 5050: Fix checkout address form, by @nihco2.

Modules page

  • 5058: Fix for module page when not on default port 80, @Quetzacoalt91.
  • 5081: Improvements for new Module Page, by @tchauviere.

Front office (default theme, named “Classic” for now, based upon the Starter Theme)

  • 5032: Improve homeslider compat with boostrap carousel and module v2.0, by @julienbourdeau.
  • 5054: Improve checkout form and replace old images, by @nihco2.
  • 5062: Added data for order confirmation page with OrderPresenter, by @mickaelandrieu.
  • 5065: shopping card redesign, by @nihco2.
  • 5066: rename default-bootstrap to classic. Thank you @fojtCz!
  • 5067: Fix ajax condition, by @maximebiloe.
  • 5075: Improved order presenter, by @mickaelandrieu.
  • 5079: My account fixes and improvements, by @maximebiloe.
  • 5083: Fix blockcart link, by @maximebiloe.
  • 5084: Improved discount amounts display, by @mickaelandrieu.

Back office

  • 5005: ThemeManager on theme upload works as expected, by @mickaelandrieu.
  • 5040: Introducing an OrderPresenter controller, by @mickaelandrieu.
  • 5055: Improved ThemeManager according to form changes, by @mickaelandrieu.
  • 5057: Synchronize stock after adding a product via ajax to an order. Thank you @Ginkosama!

Core

  • 5064: Typo in FrontController class. Thank you @gRoussac!
  • 5069: introduced core js for themes, by @mickaelandrieu.
  • 5070: Fix product order on shopping cart, by @mickaelandrieu.

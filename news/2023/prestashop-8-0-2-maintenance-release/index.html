PrestaShop 8.0.2 is available

Second maintenance version of the 8.0.x branch is available

Written by
PrestaShop team
Published

Mar 13, 2023

Share this

The second patch version for PrestaShop 8 is now available to download!

Version 8.0.2 is available! In this patch, you can find over 20 bug fixes. As a patch version, 8.0.2 brings further goodness and stability to the 8.0 branch.

There were several noteworthy issues addressed in PrestaShop’s 8.0.2 milestone. The upgrade mechanism had a problem where it couldn’t check if the PHP session had started on some webservers, which was reported by many community members. Additionally, significant problems with web services were resolved, including the inability to create new customers. Another noteworthy fix was made for enabling themes that use Symfony routes for their modules, which was reported by a community member. Finally, there were several under-the-hood fixes and a new version of Distribution API client released to enhance the module upgrade process.

It is recommended to upgrade your shop quickly in order to benefit from these fixes. Of course, don’t forget to backup before. You can upgrade to the latest version using the 1-Click Upgrade v4.15.0.

Issues involved in the milestone 8.0.2

Modules

  • It was impossible to upgrade ps_distributionapiclient module
  • Module descriptions are tagged as untranslated fields by the Symfony debugger
  • The translatable input field of the module was filled with the default language instead of being empty

Back office

  • Fixed the conversion from 3-char hex-color to 6-char hex color
  • Customer::getSimpleAddressSql() didn’t retrieve the right data. As result will be selected all the customer addresses, but not the one filtered by id_address
  • Merchant couldn’t access to the module for an employee without setting all permissions enabled
  • An exception was displayed in all migrated page after deleting language used by the employee
  • Enabling new theme was failing due to routes caching
  • The popup for data import was not well escape
  • HTML tags are not taken into account in some pages
  • An exception was thrown when import a file with non .csv extension and download the csv created file
  • The alternative currency symbol is not displayed in the product V2 & V1
  • Icon fonts were not properly preloaded in back office and it is now fixed
  • Fixed an exception when adding a bad query with an alias in a wrong position (only PHP 8+)

Front office

  • Guests were detected as a logged-in user
  • It was impossible to add a first address with error “Invalid postcode” in some circumstances

Webservices

  • The category was not well-formed from the Webservices
  • Webservice was not working properly on PrestaShop 8.0 with PHP 8.1
  • It was not possible to create new customers with the webservices

Translations

  • Translation string for “new” in BO was not working

Upgrade mechanism

  • With the 1-Click Upgrade module v4.15.0, it was not possible to create a PHP session and no new version was available

Other

  • There was a bad PHP float comparison in Tools

Download PrestaShop 8.0.2 now!

Since version 8.0.2 is a “patch” update, upgrading from version 8.0.0 should happen without any issues.

Features will work better, and modules and themes, which worked fine on previous 8.0 release, will work just as well with 8.0.2. As for every upgrade, it’s highly recommended to do a full manual backup beforehand.

If you encounter any problems during the upgrade that, in your opinion, are not caused by third-party modules or modifications used in your shop, submit a bug report.
